TRUST | Assistance 


Demande d'evolution de TRUST


Le demandeur envoie un mail au chef de projet de TRUST pour
decrir sa demande d'evolution mineure de la version officielle
de TRUST installee. Les versions beta ne seront pas evoluees. Le
demandeur est ensuite informe de la suite donnee a sa demande par
le chef de projet TRUST (barre@alpes.cea.fr).

Des remises a jour periodiques de la derniere version officielle
installee sont prevues.


Demande d'installation de TRUST


Le demandeur informe le chef de projet TRUST d'une demande
d'installation de TRUST sur une machine pour laquelle la
portabilite de Triou_U est assuree. La demande pouvant etre
assez nombreuse et la duree d'une installation (selon la version
souhaitee) varier de 2 a 5 jours, une liste de priorite
d'installation sera etablie. Il est rappele que plusieurs types
de versions peuvent etre installees :


  • Version beta (generalement la plus avancee mais en developpement
donc non maintenue)


  • Version officielle, testee et livree avec sa documentation

Et pour chacune, deux types selon deux schemas d'utilisation :


  • Version utilisateur pour realiser des calculs sans effectuer
de developpements personnels du code TRUST


  • Version developpeur pour realiser des calculs et effectuer
des developpements personnels du code TRUST

Seule la version livree sera maintenue, et aucune resolution
d'incident ne sera effectue sur une version developpee. Seule
une assistance de type co-developpement sera apportee.
