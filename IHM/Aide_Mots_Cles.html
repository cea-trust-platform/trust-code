    PrecisionGeom
    
Keyword to change the way floating-point number comparison
is done. By default, two numbers are the same if their
absolute difference is less than 1e-10. The keyword is
useful to change this value:

PrecisionGeom new_value

Moreover, nodes coordinates will be written in .geom files
with this same precision.


    Echangeur
    
Keyword to create an heat exchanger, which will
be associated to the problem. An heat exchanger
is defined thanks to several sub zones. Each sub
zone has the same number of tube per mesh element.
Each sub zone is seen as a porous media. The
parameters of the heat exchanger are:

type_pas_echangeur : Kind of exchanger, rectangular (RECT) or triangular (TRIA)
nbTubes_maille : Number of tube per mesh element
diametre_interieur : Tube internal diameter
diametre_exterieur : Tube external diameter
pasX : Distance between two tubes on x axis
pasY : Distance between two tubes on y axis
lambda_tube : Tube conductivity
virolesX : Position of ring on x axis
virolesY : Position of ring on Y axis
virolesZ : Position of ring on Z axis

There is also a keyword to change the number of tubes per
mesh element :

Modifier_nbTubes { name_of_sub_zone number } 

Porosites_echangeur_VDF is the keyword to calculate porosity for the
porous media.

Example: See the test cases:Echangeur_CRIEPI

Echangeur echangeur
Associer echangeur Suz1002
Associer echangeur pb
Lire echangeur {
        type_pas_echangeur      TRIA
        nbTubes_maille          0.25
        diametre_interieur      0.095
        diametre_exterieur      0.099
        pasX                    0.13
        pasY                    0.15
        lambda_tube             0.0006596
}
Modifier_nbTubes { Suz1002 0.5 }
Porosites_echangeur_VDF echangeur

Warning:
Heat exchanger model is available for VDF discretisation only
and is not parallelized yet.

    Standard
    
A new class, intended for LES calculations, has been developped to
optimize and parametrize most of the diffusion terms.
Keyword standard makes the new class activate. Following keywords,
described below, admit boolean values 0 or 1.

grad_Ubar 1 makes the gradient calculated through the filtered values
of velocity (P1-conform).
nu 1 (respectively nut 1) takes the molecular viscosity (eddy viscosity)
into account in the velocity gradient part of the diffusion expression. 
nu_transp 1 (respectively nut_transp 1) takes the molecular viscosity
(eddy viscosity) into account according in the TRANSPOSED velocity
gradient part of the diffusion expression.
filtrer_resu 1  allows to filter the resulting diffusive fluxes
contribution.

Remarque/recommandation : the original version : diffusion { } is
equivalent (from a theoritical point of view) to :

diffusion { standard grad_Ubar 0 nu 1 nut 1 nu_transp 0 nut_transp 1 filtrer_resu 0}.

For classical RANS calculations, user can refer directly to the former
(original) version. For LES calculations with convection { EF } in data file,
studies (in progress) have shown good results for a plane channel configuration with :

diffusion { standard grad_Ubar 1 nu 1 nut 1 nu_transp 1 nut_transp 1 filtrer_resu 0}

Example: See the test case smago_VEF_Diff_standard
diffusion  { standard grad_Ubar 1
	nu 1 nut 1 nu_transp 1 nut_transp 1
	filtrer_resu 1 }

    Modele_Rayonnement_Milieu_Transparent
    
A new possibilty is now available when reading the Radiant Model for Transparent Fluids. 
The old way to read this kind of model was to specify the name of two files : one 
defining the emicivity of radiant boundaries and the other one defining the shape factor matrix. 

Example: (this syntaxe is still available but not optimal when emissivity is constant on boundaries)
Lire mod 
{
  Test_2D_VDF_Radiation.facesrayo
  Test_2D_VDF_Radiation.factforme
}

For optimisation reasons, it appears usefull to avoid to invert the shape factors matrix at each time steap.
This new option is activated by specifing the name of the file where the inverse of the shape factors matrix
will be stored. The default format of this file is ASCCI, but if the user specifies binaire, the format 
will be binary.

Example: 
Example: See the test case Test_2D_VDF_Radiation
Lire mod 
{
  Test_2D_VDF_Radiation.facesrayo
  Test_2D_VDF_Radiation.factforme
  Test_2D_VDF_Radiation.mat_ray_inv [binaire]
}



    Tetraedriser_par_prisme
    
This new discretisation generates 6 iso-volume tetrahedral
element from primary hexahedral one (contrarily to the 5 
elements ordinarily generated by tetraedriser). This element 
is suitable for calculation of gradients at the summit 
(coincident with the gravity centre of the jointed elements 
related with) and spectra (due to a better alignment of the 
points).


    THI_NEW
    
For unstructured approach, Traitement_particulier have been slightly
modified. Averaging process respects better than previously the real 
location of the quantities (in the y-direction). Moreover, indications 
like friction velocity and reynolds, and evolution of the time step 
are calculated and written respectively in the files u_tau, 
reynolds_tau and dt_evol.
General syntax : Just substitute the previous keyword "nb_int" and 
"dir_echant" to the following ones : Ny and eps. Ny is set to initialise 
the dimension of tables and eps is the tolerance to check if points 
belongs to the same y-co-ordinate of the other points.
Note that the present post-treatment is suitable only for plane 
channel configuration with wall normal direction according to y.

Example:
Traitement_particulier
{
	THI_new
	{
		init_Ec 1 val_Ec 1.5 facon_init 0
   		calc_spectre 1
	}
}


    Lecture_Sous_Zone
    
New keyword to affect processors to specified sub-zones.
general syntax is :
LECTURE_SOUS_ZONE Nbssz sous_zone1 sous_zone2
The number of sub-zones (Nbssz ) has to be similar to Nb_parts, or 
the difference Nb_parts- Nbssz has to be set equal to 1 in order the 
unaffected elements (out of sub-zones) are automatically associated to the last, free processor.

Example: See the test case Courants_paras_perio_SG

Decouper dom
{
        Pmetis
        Nom_Zones dom_par
        Larg_joint 1
        Nb_parts 3
        LECTURE_SOUS_ZONE 2 sous_zone1 sous_zone2
}


    THI_thermo
    
A new Treatment for the temperature field is available for THI computation.
It offers the possibility to :
- evaluate the probability density function on temperature field,
- gives in a file the temperature field for a future spectral analysis
- monitor the evolution of the max and min temperature on the whole domain

The syntax is the same than for special treatment THI.

Example: See the test case TP_THI_ThHyd_centre_perio_VDF

Traitement_particulier { THI_thermo { init_Ec 1 val_Ec 1.5 facon_init 0
                                               calc_spectre 1 } }

    Paroi_Knudsen_non_negligeable
    
New boundary condition for number of Knudsen (Kn) above 0.001 where slip-flow
condition appears: the velocity near the wall depends on the shear stress :
Kn=l/L with l is the mean-free-path of the molecules and L a characteristic length scale.

U(y=0)-Uwall=k(dU/dY)

Where k is a coefficient given by several laws:
Mawxell: k=(2-s)*l/s
Bestok&Karniadakis:k=(2-s)/s*L*Kn/(1+Kn)
Xue&Fan:k=(2-s)/s*L*tanh(Kn)

s is a value between 0 and 2 named accomodation coefficient. s=1 seems a good value.

}

    Schema_Crank_Nicholson
    
The Crank-Nicolson method is of second order accuracy. It is based either
on the trapezoide rule or the mid-point rule. Here the mid-point rule
formulation is used (Euler-centered scheme).

The basic scheme is: u(t+1) = u(t) + du/dt(t+1/2)*dt.

The time derivative du/dt at the level (t+1/2) is calculated iteratively
with a simple under-relaxations method. For the method is implicit, neither
the cfl nor the fourier stability criteria must be respected. The time step 
is calculated in a way that the iterative procedure converges in 5 iterations.
Parameter:
omega: relaxations factor
seuil: accuracy of the solution

syntax:

solveur_iteratif 
 {
    omega 0.5
    seuil 0.1
 }

The time derivative du/dt at the level (t+1/2) is calculated by means of 
a non-linear gmres solver. The non-linear GMRES is Newton method where the 
derivative is calculated numerically and the resulting linear system is 
solved by a GMRES method. For the method is implicit, neither the cfl nor 
the fourier stability criteria must be respected. The time step is calculated 
in a way that the Newton method converges in 5 iterations.

Parameter:
nkrilov: number of krilov vectors
seuil: accuracy of the solution 

syntax:

solveur_gmres 
 {
    nkrilov 2
    seuil 0.1
 }

Example: See the test cases ThHyd_CrankNicholsonImpliciteIteratif and ThHyd_CrankNicholsonImpliciteGmres

Schema_Crank_Nicholson sch
Lire sch
{
        nb_pas_dt_max 3
        tinit 0.
        tmax 1.
        dt_min 0.001
        dt_max 0.2
        dt_impr 0.001
        dt_sauv 100
        seuil_statio 1.e-8
        facsec 1.1
        solveur_gmres {
        seuil 0.1
        nkrilov 4
        }
}

    Debog
    
Keyword to debug some differences between two TRUST versions on a same data file.

Debog problem_name file_to_write_domain file_to_write_faces error mode_debog

problem_name : Name of the problem to debug
file_to_write_domain : Name of the file where domain will be written in sequential calculation
file_to_write_faces : Name of the file where faces will be written in sequential calculation
error : Minimal value (by default 1.e-20) for the differences between the two codes  
mode_debog : By default -1 (nothing is written in the different files), you will set 0 for
		the run with the first code, and 1 for the run with the second code.
		
If you want to compare the results of the same code in sequential and parallel calculation,
first run (mode_debog=0) in sequential mode (the files file_to_write_domain and file_to_write_faces
will be written first) then the second run in parallel calculation (mode_debog=1).
		
During the first run (mode_debog=0), it prints into the file DEBOG, values at different
point of the code thanks to the C++ instruction call (see for example in Noyau/Resoudre.cpp file): 

Debog::verifier(msg,value);

Where msg is a string and value may be a double, integer or array.

During the second run (mode_debog=1), it prints into a file Err_Debog.dbg the same messages
than in the DEBOG file and checks if the differences between results from the two codes are
less than error. If not, it prints Ok else show the differences and the lines where it 
occured.

Example:
dimension 2
Pb_Thermohydraulique pb
...
Discretiser pb dis
Debog pb seq faces 1.e-6 0
Lire pb { ... }
Resoudre pb

Warning:
Debog doesn't work well with several coupled problems.

For a parallel calculation, it is the keyword which should replace the Raccord local homogene

    mpirun
    
Command line to run a parallel calculation.

First the data file (ex: study.data) must contain directive Decouper to partition the domain and the TRUST binary
must run in sequential mode.

$ $exec study 1>out 2>err

Then, once the files containing the partitions are generated, change the data file to add the Scatter directive
to read theses files. Then we can run TRUST in parallel mode thanks to mpirun:

$ mpirun -np n $exec study n 1>out 2>err

n is the number of processors which should match the number of partitions.


    Decouper Scatter
    
These keywords are used for parallel calculation to cut a domain for each processor and to after to read the partionned mesh.
By default, theses keywords are commented in the reference test cases.

Decouper dom: To cut the domain dom. This keyword has the following options:
PMetis: Partionned by the tool Metis. Must be used for unstructured meshes.
Tranche nx ny nz: Partionned by nx in the X direction, ny in the Y direction, nz in the Z direction. Works only for structured meshes.
Nb_parts n: Number of partitions in the domain. If keyword Tranche is used, n=nx*ny*nz
Larg_joint 1: Length of the covering mesh of one processor above another one. It depends of the convective scheme:
		Generally 1 (amont, centre), sometime 2 (quick in VDF).
Nom_Zones name: Name of the files containing the different partition of the domain.

Scatter name.Zones dom: Keyword to read the partitions of the domain dom in the files called name_000n.Zones

So a parallel calculation leads to two steps (the first in sequential mode to prepare the partition, the second in parallel mode):
a) Comment the data file to read the mesh and partition it with Decouper.
b) Comment the data file to run calculation by reading the partitionned mesh with Scatter.

Example: See the test cases:M3DLAM.
dimension 3
domaine dom
# DEBUT MAILLAGE #
Lire_fichier M3DLAM.geo ;
# FIN MAILLAGE #

# DEBUT DECOUPAGE
Decouper dom
{
        PMetis
        Nb_parts 2
        Larg_joint 1
        Nom_Zones DOM
}
Fin
FIN DECOUPAGE #

# DEBUT LECTURE
Scatter DOM.Zones dom
FIN LECTURE #

    DI_L2

    Diffusion_implicite
    
Mot cle pour impliciter le terme de diffusion dans l'equation de Navier
Stokes. Le pas de temps de stabilite est desormais base uniquement sur
le pas de temps de convection (dt=facsec*dt_convection). On gagne ainsi,
dans certains cas (diffusion grande par rapport a la convection sur des
maillages resserres) un facteur important sur le pas de temps.

Exemple: (Voir aussi le cas test defilante_P1NC_diff_impl)
Schema_Euler_Explicite sch
Lire sch
{
        tinit 0.0
        tmax 0.01
        dt_min 1.e-5
        dt_max 0.001
        dt_impr 1.e-6
        dt_sauv 100
        seuil_statio 1.e-5
        facsec 10.
	diffusion_implicite 1
}
Attention:
- Il est tout de meme conseille de ne pas depasser avec un facsec trop grand
le pas de temps de convection du calcul. Debuter avec un facsec de 1 puis
augmenter progressivement si vous voulez accelerer le calcul.
- En outre, pour un calcul de convection naturelle avec une vitesse initiale
nulle, au premier pas de temps, le temps de convection est infini et donc
dt=facsec*dt_max.
- Attention egalement au message suivant dans le fichier d'erreur :
"Non convergence diff_impl en : 101 iterations
 Residu : 258.53
 solution continue avec d(unconnu)/dt = 0"
L'inconnue au pas de temps (n+1) est alors prise egale au pas de temps (n)
et il peut etre plus prudent d'arreter le calcul et de repartir en 
baissant le facsec.

    Schema_Euler_Implicite
    
Mot cle pour designer un nouveau schema en temps de type implicite. Les
operateurs de convection et de diffusion des equations sont implicites.
Pour accroitre le pas de temps (base par defaut sur le calcul du pas de temps
de stabilite), augmenter la valeur du facteur de securite facsec. Les algorithmes
 disponibles en VDF et VEF sont de type SIMPLER (mots cles Simpler_VDF Simpler_VDF).
Ils necessitent la definition de : 

seuil_convg_implicite critere de convergence de la methode SIMPLER. On peut conseiller
de le fixer a un millieme de l'ordre de grandeur de la vitesse dans un premier temps
max_iter_implicite nombre maximum d'iterations pour le solveur GMRES (fixe au minimum a 10).
On peut conseiller de le fixer a 1 dans un premier temps.
seuil_resol_implicite critere relatif de convergence pour le solveur GMRES (inversion des matrices).
 On peut conseiller de le fixer a 10e-8 dans un premier temps.

Exemple:  (Voir aussi les cas tests sans_impl VAHL_DAVIS_IMPL docond_impl)
Schema_Euler_Implicite sch
Lire sch {
	tinit 0.
	tmax 1000.
	dt_min 0.0001
	dt_max 0.1
	dt_impr 0.0001
	seuil_statio 0.1
	facsec 10
	Solveur Simpler_VDF
	{
		max_iter_implicite 50
		seuil_convg_implicite 0.001
		seuil_resol_implicite 1.e-8
	}
}
Attention:
- En VDF, l'algorithme du Simpler applique a l'equation de Navier Stokes n'est code que
pour le schema amont.
- En VEF, l'algorithme du Simpler n'est pas valide.
